# Workflow

Core workflow:

Lead → Hypothesis → Code Review → PoC → Evidence Gate → Report Gate → Submit / Kill / Archive

1. **Lead** — identify a signal or bug idea.
2. **Hypothesis** — write the exploit sentence and expected impact.
3. **Code review** — confirm reachability and kill conditions.
4. **PoC** — write a local exploit test and control test when practical.
5. **Evidence gate** — verify the PoC, assertions, impact, and artifacts.
6. **Report gate** — run the 7-question readiness check.
7. **Submit / Kill / Archive** — submit only strong findings; otherwise preserve the decision.

Most leads should be killed, downgraded, or archived. The goal is fewer weak reports and stronger evidence.
